prefix_256 MeanR — 无PCA基线 vs 各PCA方案 基线全程领先
关键观察：无PCA基线（绿色粗线）在所有12层上全面领先所有PCA方案。差距在浅层（Layer 1-4）就已形成，深层持续扩大。PCA3 最接近但始终低于基线，PCA4 始终最差。
full_512 MeanR — 无PCA基线 vs 各PCA方案
信息补偿效应：PCA1 的 full_512 在深层（Layer 11-12）反超基线（74.69 vs 74.39）。这是因为 PCA1 的 prefix_256 被限制后，模型将更多有用信息编码到 suffix（后256维）进行补偿。full_512 的微小优势并不代表 PCA 有效——它恰好说明模型在弥补 PCA 造成的 prefix 损失。

info_64 高 ≠ 整体好：PCA1（49.24）和 PCA4（46.32）的 info_64 远高于无PCA基线（38.00），但 prefix_256 反而更低。原因：PCA 将方差集中到 info_64，代价是 task_192 通道质量下降。无PCA基线的 info_64 虽然最低，但 info 和 task 的协同效果最好，prefix_256 整体最高。

五、PCA2 后处理投影分析

无PCA基线 vs PCA2 eval_pca 后处理 后处理 PCA 破坏已学结构
核心发现：PCA2 的训练阶段与无PCA基线完全相同（数据已验证一致）。训练后在 ESE 特征上做 PCA 投影，prefix_256 从 67.17 跌到 60.54（-6.63）。说明 ESE 模型已经学到了比 PCA 更优的特征分配方式，后处理 PCA 投影只会破坏已学结构。

八、为什么 PCA 方向无效？

根因分析

核心矛盾：PCA 优化方差 ≠ 优化检索

PCA 是无监督线性变换，目标是最大化投影后的方差。但 ESE-CLIP 的目标是跨模态检索（cross-modal retrieval），需要的是语义对齐而非方差最大化。高方差维度可能只是噪声或模态内方差，与跨模态匹配无关。

具体失败模式

PCA方案失败原因
PCA1 固定投影在 Raw CLIP 空间拟合的 PCA 与 ESE 学到的特征空间不匹配。固定投影矩阵无法随训练更新，约束了 info 通道的学习自由度
PCA2 Eval投影ESE 已通过 split loss 学到了最优的 info/task 分配。PCA 投影强行重新分配信息，破坏了已学到的跨通道协同结构
PCA3 辅助Loss软约束方向正确（不强制投影），但 PCA 目标与检索目标不对齐，辅助 loss 实际上在将 info_64 拉向次优方向。接近但不超过基线
PCA4 完整旋转最激进也最失败。完整旋转后前64维获得最大方差（info_64↑），但后192维只剩低方差残差（task_192↓↓），总体 prefix 和 full_512 双降

无PCA基线为什么最好？

  1. 端到端优化：split loss 直接针对检索目标优化 info/task 分配，学到了任务相关的最优分割方式
  2. 非线性特征：ESE 模型能学到 PCA 无法表达的非线性特征组合
  3. 通道协同：info_64 虽低（38.00），但与 task_192（62.48）协同后 prefix_256 最高（67.17），说明两个通道的信息互补性最好
  4. 压缩效率最高：256维捕获了512维 90.3% 的检索能力，PCA 方案均更低

九、最终结论

PCA 方向应终止

4种不同的 PCA 引入方式（训练前固定、训练后投影、训练中辅助、端到端旋转），无一超过无PCA基线。这不是调参能解决的——是 PCA 假设本身与检索任务不兼容。

当前最优：无PCA 的 Feature Split v2

  • prefix_256 MeanR 67.17%
  • full_512 MeanR 74.39%
  • 压缩效率 90.3%（256维/512维）

建议下一步方向（非PCA）

  • 提升 full_512 上限：当前 74.39%，提升它也会带动 prefix_256
  • 缩小 512→256 差距：当前差距 7.22 pt，可探索更好的 split loss 或可学习投影
  • 模型架构改进：增加 ESE 模块容量、多层特征聚合等
  • 如果仍想压缩 info 通道：用可学习的线性投影（端到端训练）替代固定 PCA
